# Multilingual text evaluation

`evaluation/phase4_multilingual_text.jsonl` is a curated synthetic dataset with 360 scenarios: 90 each for `zh-CN`, `yue-Hant-HK`, `en-HK`, and `mixed`. Each group contains 45 semantic categories with two reviewed paraphrases. The second paraphrase changes syntax, verb, request framing, entity order, quantity construction, alias, or code-switch position; punctuation, whitespace, case, width, courtesy-only, and sentence-final-particle-only variants are rejected. The catalog has 64 designed ambiguity/clarification scenarios and 128 HANDOFF/REFUSE scenarios.

The validator uses NFKC, casefolding, punctuation removal, whitespace normalization, and conservative removal of pure courtesy prefixes/suffixes. Its current gate is surface unique 360/360, normalized unique 360/360, near-duplicate groups 0, punctuation/spacing/case duplicate groups 0, and 45 base categories per group. Mixed rows explicitly cover Chinese-, Cantonese-, and English-dominant input; English verbs with Chinese dishes; Chinese verbs with English dishes; Cantonese modifiers with English items; English quantities with Chinese units; and Chinese quantities with English units.

The runner defaults to `--mode both` and also supports `--mode auto` and `--mode assisted`.

- Auto sends no `locale`, `localeHint`, or `localeLocked` for either setup or evaluated turns. Dataset locale fields are comparison-only ground truth. Current main-set results are auto locale 360/360, auto dominant locale 270/270 where annotated, auto response locale 360/360, intent/classification/mutation 360/360, item 120/120, quantity 72/72, and modifier 8/8.
- Assisted models a concrete UI language choice by passing only `locale=<concrete locale>` and `localeLocked=true`. It never passes `mixed` as a reply locale and never duplicates the choice in `localeHint`. Current results are assisted response locale 360/360 and assisted intent 360/360; item, quantity, modifier, classification, and mutation retain their independent denominators.

`evaluation/phase4_locale_detection.jsonl` adds 160 no-hint text cases (40 per group), including single menu names, lock transitions, numbers/punctuation, unsupported scripts, explicit switches, traditional standard Chinese, mixed dominance, English abbreviations, Hong Kong service English, Cantonese colloquial markers, and Chinese text with English brand terms. Exact detection is 156/156. Four inherently ambiguous text-only cases are excluded from exact-locale accuracy and pass conservative handling 4/4. Unsupported handling is 4/4.

All expected fields remain outside runtime kwargs. Evaluator tests change or remove expected intent/locale labels and verify parser inputs and outputs are unaffected. The runner is offline and keeps all LLM/voice/TTS flags disabled.

Specialized metrics use their own denominators. Current results are HANDOFF reason 64/64, REFUSE reason 64/64, confirmation 24/24, handoff false positives 0, refusal false positives 0, wrong mutations 0, confirmation bypasses 0, serious-allergy omissions 0, fake merchant acceptance 0, and live-LLM triggers 0. A zero denominator is reported as `not_evaluated`, never as 100%.

`duplicate_order_line_items` only checks repeated cart line identity. Database checks query `ConversationSession`, `Order`, `OrderConfirmation`, and `IdempotencyRecord` with SQL after every scenario. Current results are database order counts 360/360, duplicate order lines 0, duplicate database orders 0, duplicate active confirmations 0, and duplicate idempotency records 0. Explicit confirmation scenarios create one real synthetic database order and one active confirmation.

Cross-tenant refusal classification and real isolation are separate. Text classification checks 16 relevant cases per mode with 0 errors. The reusable SQLite/PostgreSQL access audit performs 10 real checks covering restaurant and branch session rebinding, order lookup, handoff lookup, response disclosure, lexicon/menu-version scope, and no-write invariants; current leak failures are 0.

These are deterministic results on reviewed synthetic text. They are not estimates of real calls, speech, accents, arbitrary Hong Kong colloquial language, open-domain translation, or order correctness. Language confidence describes only the detector. Cantonese coverage remains finite, and pure traditional standard Chinese may not reliably distinguish Mandarin from Cantonese without session context or user choice.

# 30–60 秒公开演示脚本

建议先启动纯文本 demo。语音能力依赖本机模型、麦克风和播放设备，可作为第二段展示。

## 录屏流程

依次输入：

1. `推荐一下`
2. `第一个来一份`
3. `再来一份可乐`
4. `把可乐删掉`
5. `我要配送到中山大学南校园，电话是 13800000000`
6. `确认订单`

画面重点：推荐候选、统一订单摘要、菜品增删、配送信息、手机号掩码、确认后订单号与 `mock order` 提示。

地址和手机号均为演示示例，不对应真实个人。最终提交只创建内存 mock order，不涉及支付、真实商户或真实配送。

## 录制建议

- 从“新订单”开始，保持调试信息和语音诊断折叠。
- 首次演示优先使用文本路径，确保节奏稳定。
- 若录制语音，先按 [语音演示与排障](voice-troubleshooting.md) 完成本机检查。
- 不在录屏、终端或浏览器中展示 `.env`、API key 或真实个人信息。
